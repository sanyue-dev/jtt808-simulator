# 创建行程并用 DevTools 监控调试

使用 Chrome DevTools MCP 完成以下流程。参数从用户输入获取，如未提供则使用默认值。

## 参数

- `$ARGUMENTS` 格式：`线路名 车牌号 终端ID SIM卡号 端口`
- 默认值：
  - 线路：东巴凤环线公交
  - 车牌号：桂M88888
  - 终端ID：DBF00001
  - SIM卡号：013800008888
  - 端口：7100

## 步骤

### 1. 确保应用运行
- 检查 `http://127.0.0.1:18888/task/index` 是否可达
- 如不可达，用 `mvn spring-boot:run` 后台启动，等待就绪

### 2. 打开任务创建页
- 用 Chrome DevTools 导航到 `http://127.0.0.1:18888/task/index`
- 抓取页面快照，确认表单元素可用

### 3. 填写并提交表单
- 选择指定线路（从下拉框匹配）
- 填入车牌号、终端ID、SIM卡号
- 确认端口字段值为用户指定端口
- 点击"启动"按钮
- 检查 XHR 响应：查看 `POST /task/run` 的网络请求响应体，`error.code === 0` 表示成功
- 提示：页面使用自定义 toastr 通知（动态 DOM 元素），不使用浏览器原生 alert

### 4. 进入监控页
- 导航到 `http://127.0.0.1:18888/monitor/list/index`
- 等待车牌号出现在列表中
- 快照确认新任务在列表中，从"详情"链接 URL 中提取任务 ID（格式 `?id={taskId}`）

### 5. 打开地图监控详情
- 点击"详情"链接（注意：该链接有 `target="_blank"`，会打开新标签页）
- 用 `select_page` 切换到新打开的详情页标签
- 等待"行程任务详情"文本出现，确认页面加载完成
- 从快照中读取并向用户汇报当前任务状态（线路、坐标、最后上报时间）

### 6. 检查控制台
- 列出页面 console 错误和警告
- 如有报错，分析原因并告知用户
